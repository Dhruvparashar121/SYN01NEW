What is the Framework?

This lab shows how traffic moves from a Kali client through controlled relay stages before reaching the final IT network.

  • Client: the Kali machine where testing commands are launched.
  • Firewall: controls which traffic is allowed to enter or leave the lab path.
  • VPS-1 & VPS-2: relay nodes that forward traffic and change what each side can directly observe.
  • Tor / final stage: represents the last anonymity or observation layer in the training chain.

Why it Matters

Students must understand both sides — what the red team sends and what the blue team can actually see at each hop.

  • Red-team view: route validation, tunnel checks, traceroute testing and leakage checks.
  • Blue-team view: observe packets, source-IP changes, ports, DNS/ICMP evidence and timing.
  • Instructor view: every hop with repeatable commands, expected outputs and live screen evidence.
  • Safety: keep the activity inside an authorised Proxmox lab/VLAN only.

    Beginner End-to-End Configuration Guide

    1. Install WireGuard: sudo apt update && sudo apt install wireguard -y
    2. Generate WireGuard keys: wg genkey | tee privatekey | wg pubkey > publickey
    3. Generate SSH key: ssh-keygen -t ed25519 -C labuser
    4. Copy SSH key to VPS: ssh-copy-id user@VPS_IP
    5. Create /etc/wireguard/wg0.conf on client and server.
    6. Enable forwarding: sudo sysctl -w net.ipv4.ip_forward=1
    7. Start tunnel: sudo wg-quick up wg0
    8. Verify: sudo wg show and ping peers.
    9. Install Tor + Proxychains if required.
    10. Validate route using traceroute and Wireshark.
