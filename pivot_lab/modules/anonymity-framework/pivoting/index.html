01

Pivot Host

The compromised host has dual visibility: it can communicate outward to the operator and inward to a hidden subnet. It becomes the relay point, not the final objective.

02

Route Control

Autoroute inserts knowledge into Metasploit’s route table so MSF modules understand which session can reach the internal network.

03

SOCKS Bridge

A local SOCKS listener on the attacker side exposes the tunnel to external tools. It is usually reached as 127.0.0.1:1080.

04

Internal Target

The final objective is simplified to an internal target. Learners validate reachability through the pivot route without showing any separate listener concept.

      Internal target workflow
      
          
      1Is the internal target directly reachable from the external operator?No → direct access fails because segmentation blocks the path.
      
          
      2Can the operator reach the internal target through the pivot route?Yes → traffic reaches the target through SOCKS, Metasploit route and the pivot host.
      
          
      3Which path generates defender evidence?The evidence is the routed path: proxy use, pivot-host traffic, and internal service access.
      
          
      4What should the learner conclude?Successful access is proven by route validation and target response, not by placing another listener in the diagram.

      Exam clarity matrix
      
        
      Route, SOCKS, proxychains and pivot host are different things
      
      
      
      
      
        
      ElementWhere it existsWhat it doesCommon mistake removed
      
        
      Pivot hostCompromised perimeter/DMZ hostRelays traffic into a hidden subnet because it has internal visibility.It is not the SOCKS listener.
      
        
      AutorouteMetasploit route tableMaps an internal subnet to a specific Meterpreter session.It does not change the attacker OS route table.
      
        
      SOCKS proxyAttacker side local listenerAccepts TCP connections and sends them down the Meterpreter tunnel.127.0.0.1:1080 is normally on the attacker machine.
      
        
      ProxychainsAttacker-side wrapperForces external programs to use the SOCKS listener.It does not create the tunnel by itself.
      
        
      Internal targetHidden subnet endpointFinal reachable system after the pivot route and SOCKS path are validated.It is the objective, not a separate listener box.
